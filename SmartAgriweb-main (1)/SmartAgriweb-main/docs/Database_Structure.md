# SmartAgriWeb — Database Structure

Database: SQLite (`backend/database/smartagri.db`)

---

## Tables

### `sensor_readings`
Stores all sensor data (from ESP32 or simulated).

| Column | Type | Description |
|--------|------|-------------|
| id | INTEGER PK | Auto increment |
| temperature | REAL | °C |
| humidity | INTEGER | % |
| soil_moisture | INTEGER | % |
| light_lux | REAL | Lux value from BH1750 |
| source | TEXT | `'esp32'` or `'simulated'` |
| recorded_at | TIMESTAMP | Auto-set on insert |

---

### `alerts`
Alerts generated by the alert engine.

| Column | Type | Description |
|--------|------|-------------|
| id | INTEGER PK | Auto increment |
| type | TEXT | `'warning'`, `'danger'`, `'info'` |
| title | TEXT | Short alert title |
| message | TEXT | Full alert description |
| resolved | INTEGER | 0 = active, 1 = resolved |
| created_at | TIMESTAMP | When alert was created |

---

### `active_crops`
Tracks which crops are currently growing.

| Column | Type | Description |
|--------|------|-------------|
| id | INTEGER PK | Auto increment |
| crop_key | TEXT | Key from crop dataset (e.g. `'tomato'`) |
| field_name | TEXT | Farm field identifier |
| planted_at | DATE | Planting date |
| status | TEXT | `'growing'`, `'harvested'`, `'failed'` |
| notes | TEXT | Farmer notes |
| created_at | TIMESTAMP | Record created |

---

### `irrigation_log`
Daily irrigation records.

| Column | Type | Description |
|--------|------|-------------|
| id | INTEGER PK | Auto increment |
| liters_used | REAL | Liters irrigated |
| liters_wasted | REAL | Estimated waste |
| crop_key | TEXT | Target crop (nullable) |
| started_by | TEXT | `'manual'` or `'auto'` |
| logged_at | TIMESTAMP | When irrigation occurred |

---
